# 五级流水线CPU项目 - ORI指令实现

## 项目概述

本项目是一个基于Verilog语言实现的五级流水线CPU设计，支持MIPS指令集中的ORI（按位或立即数）指令。该设计采用经典的五级流水线结构，包括取指(IF)、译码(ID)、执行(EX)、访存(MEM)、写回(WB)五个阶段。

---

## 文件说明

### 核心模块

#### `defines.v` - 全局常量定义
- **作用**: 定义项目中使用的所有宏常量，包括操作码、控制信号、总线宽度等
- **关键定义**:
  - 字长: 32位
  - ORI指令的操作码: `6'b001101`
  - ALU操作类型: NOP、OR、ORI等
  - 寄存器数量: 32个32位寄存器
  - 指令存储器大小: 128条指令

#### `top_file.v` - 顶层模块
- **作用**: 连接CPU和指令ROM，是整个项目的入口点
- **输入**: 时钟信号(clk)、复位信号(rst)
- **功能**: 实例化cpu模块和inst_rom模块，并建立它们之间的连接

#### `cpu.v` - CPU顶层模块
- **作用**: 实现五级流水线CPU的总体设计
- **功能**: 
  - 实例化所有流水线阶段模块
  - 连接各阶段之间的信号通路
  - 协调指令从取指到写回的全过程

### 取指阶段(IF - Instruction Fetch)

#### `pc_reg.v` - 程序计数器模块
- **作用**: 存储和管理程序计数器(PC)的值
- **功能**:
  - 每个时钟周期将PC递增4(每条指令4字节)
  - 提供ROM芯片使能信号控制
  - 支持复位功能

#### `inst_rom.v` - 指令只读存储器
- **作用**: 存储CPU要执行的指令
- **容量**: 128条指令(可根据需要扩展)
- **功能**: 根据地址输出对应的32位指令数据

### 译码阶段(ID - Instruction Decode)

#### `inst_decode.v` - 指令译码模块
- **作用**: 解析指令，生成控制信号和操作数
- **功能**:
  - 识别指令类型(目前支持ORI指令)
  - 提取立即数和寄存器地址
  - 生成ALU操作码和操作选择信号
  - 控制寄存器读取

### 执行阶段(EX - Execute)

#### `ex.v` - ALU执行模块
- **作用**: 执行算术逻辑运算
- **功能**:
  - 执行逻辑运算(目前支持OR操作)
  - 将运算结果准备好送至写回阶段

### 访存阶段(MEM - Memory Access)

#### `memory.v` - 内存访问模块
- **作用**: 处理内存读写操作
- **功能**: 目前主要是传递数据(为后续扩展预留接口)

### 写回阶段(WB - Write Back)

#### `write_back.v` - 写回模块
- **作用**: 将执行结果写回到寄存器文件
- **功能**: 将ALU运算结果或内存读取数据写入目标寄存器

### 数据通路

#### 通用寄存器文件

`regfile.v` - 寄存器组模块
- **作用**: 存储CPU的32个通用寄存器
- **容量**: 32个寄存器，每个32位
- **功能**:
  - 支持同时进行两路读操作
  - 支持一路写操作
  - 寄存器$0始终为0
  - 支持复位初始化

### 流水线寄存器

这些模块用于在流水线的各个阶段之间传递和锁存数据：

#### `if_id.v` - IF/ID流水线寄存器
- **作用**: 提取指令和PC从IF阶段传递给ID阶段

#### `id_ex.v` - ID/EX流水线寄存器
- **作用**: 将ALU操作码、操作数、目标地址等信息从ID阶段传递给EX阶段

#### `ex_mem.v` - EX/MEM流水线寄存器
- **作用**: 将执行结果从EX阶段传递给MEM阶段

#### `mem_wb.v` - MEM/WB流水线寄存器
- **作用**: 将最终结果从MEM阶段传递给WB阶段

### 测试模块

#### `tb_ori.v` - ORI指令测试台
- **作用**: 测试整个CPU设计的功能
- **功能**:
  - 生成50MHz的时钟信号(周期10ns)
  - 加载4条ORI测试指令到inst_rom中
  - 打印调试信息，显示PC、指令等执行情况
  - 在仿真完成后打印寄存器最终值

## 使用指南

### 仿真步骤

1. **修改指令**
   - 打开 `inst_rom.v` 文件
   - 在 `initial` 块中找到指令初始化部分
   - 根据需要修改或添加汇编指令的机器码
   - 示例: `inst_mem[0] = 32'h34011100;` 表示 ORI $1, $0, 0x1100

2. **运行仿真**
   - 使用Verilog仿真工具(如Vivado等)
   - 编译设计文件和测试台
   - 根据需要配置仿真时长(默认1000ns)

3. **查看仿真结果**
   - 观察控制台的打印输出，包括PC、指令执行情况
   - 审查寄存器文件最终的值
   - 验证执行结果是否符合预期

### 指令格式

ORI指令的机器码格式(32位):
```
[31:26] = 001101  (ORI操作码)
[25:21] = rs      (源寄存器地址)
[20:16] = rt      (目标寄存器地址)
[15:0]  = imm     (16位立即数，高位补零后与rs按位或)
```

### 添加新指令

要支持新的指令类型：
1. 在 `defines.v` 中定义新指令的操作码和ALU操作
2. 在 `inst_decode.v` 中添加译码逻辑
3. 在 `ex.v` 中实现对应的ALU操作
4. 在 `tb_ori.v` 中添加测试用例

## 流水线工作原理

### 执行过程

每条指令依次经过5个阶段：

1. **IF阶段**: PC从inst_rom中取出指令
2. **ID阶段**: 将指令译码，从regfile中读取操作数
3. **EX阶段**: ALU执行逻辑操作
4. **MEM阶段**: 处理内存操作(目前未使用)
5. **WB阶段**: 将结果写回regfile

### 典型时序

- 时钟周期: 10ns (100MHz)
- 每条指令耗时: 5个周期(初始延迟) + 1个周期(后续指令)
- 复位时长: 195ns
